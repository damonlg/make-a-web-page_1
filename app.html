Lesson 2

Code

In order for developers and maintainers to easily identify what performs what function, it is important for programs to maintain a logical structure. In HTML the structure is a series of nested boxes that form the tree-like structure. A more in-depth discussion of code structure can be read here.

Code is a language and therfore has syntax and rules.
Everything on a webpage is a box, putting everything in a box helps seeing the we as code and code as web.

Lesson 3

CSS

When writing code it's important to first get the structure right (boxes), then tags, then size of boxes, then position of the boxes. See the order below.

1. Look for natural boxes
2. Look for repeated styles and semantic elements
3. Write your HTLM
4. Apply styles (biggest to smallest)
5. Fix things

Repetion

Avoiding Repetition is very important for a number of reasons.

Most importantly it avoids errors. Copying and pasting errored code could lead to a lot of lost man hours and a lot of frustration!

Also can help with consistency in creating and changing page style.
